# Karen Content Agent — Master Handoff

עודכן: 23.7.2026 (זיהוי סטטוס גמיש, שפה נקייה, יוזמה של הסוכנת)
מסמך זה הוא המקור היחיד והקובע. הוא מחליף את כל גרסאות ה-Handoff הקודמות. ההיסטוריה המפורטת נמצאת בגיט וב-docs/. במקרה של סתירה: הקוד העדכני קובע.

## ויז'ן

יוצרת תוכן מתעוררת בבוקר, פותחת וואטסאפ, והסוכן כבר יודע מה יש לה, מה תקוע, מה צריך לקרות היום — בלי שהיא תפתח גיליון אחד. לטווח ארוך: כלי למנהלי סושיאל מרובים דרך וואטסאפ בלבד. הממשק הוא WhatsApp; Google Sheets הוא הזיכרון.

**שלב נוכחי — השקה מחדש (12.7.2026):** כל התקופה עד 12.7 (קפריסין/חתונה) מוגדרת רטרואקטיבית כעידן טסטים. הדאטה שלה אורכב בעותק גיליון נפרד ("היסטוריית טסטים"), הגיליון החי אופס, וקרן מתחילה שימוש אמיתי. **מדד העל: 2 רילסים אורגניים בשבוע.** הנחיה לשבועיים הקרובים: לא לגעת בקוד — לאסוף שימוש אמיתי ו-Routing Traces.

## Stack טכני

- Node.js + TypeScript + Express

- Claude API — שני מודלים: `CREATIVE_MODEL` (ברירת מחדל claude-sonnet-4-5) ליצירה/עריכה, `CLASSIFIER_MODEL` (ברירת מחדל claude-haiku-4-5) לסיווג ו-matching

- ngrok לטסטינג מקומי
- GitHub: eitanharpaz-code/karen-content-agent

## מודל התוכן (החלטות מוצר 12.7)

- **קטגוריה**: תמיד "כללי" לרעיונות חדשים (פרומפט הדראפט קובע; עריכה משמרת קטגוריה קיימת). באג הקטגוריות ההארד-קודד — לא רלוונטי עוד.
- **ההבחנה שחשובה**: אורגני מול ממומן — עמודה H, מזוהה אוטומטית מטקסט הרעיון+הסיכום. אוצר מילים: שת"פ/שתפ, שיתוף/שיתופי פעולה, חסות, ממומן/ת, ברנד, מותג, לקוח, קולאב, collab. lookahead מונע false positive על "משתפת/משתפים".
- **יעדים שבועיים** (planning-health): 2 רילס + 1 פוסט אורגניים. שת"פים לא נספרים ליעד.

## זרימות עובדות (תמצית)

רעיון חדש → דראפט → עריכה → אישור → בנק רעיונות. "תוסיפי את X להפקה" → תכנים שאושרו + משימת הפקה. עדכוני סטטוס. גאנט: שאילתות, שיבוץ עם בדיקת התנגשות, Monthly Planning. ארכיון: העברה/צפייה/שחזור. בדיקת כפילות (זווית ספציפית, לא נושא). Daily Brief 08:55 + תזכורת 17:00. "מה דחוף". **חדש (12.7): שורת סטטוס שבועי בבריף הבוקר** ("עוד לא עלה ריל השבוע / עלה אחד מתוך 2 / היעד הושג") **+ שורת הכרה בתזכורת הערב כשהיעד הושג.**

## כללי ארכיטקטורה קריטיים

- **מנוע התעדוף (P0–P4) הוא הסמכות** — גאנט-first. שכבת המדד השבועי היא הקשר מוטיבציוני בלבד ואסור שתשנה בחירת פוקוס.
- **visibility נבדק לפני production updates**; מסווג ה-AI מדולג (skipAI) כשפקודה דטרמיניסטית תתפוס.
- **כל mutation דורש אישור מפורש של קרן.**
- **State פרסיסטנטי (Stage G) + TTL 3 שעות** על מצבים ממתינים.
- **הפרדת persona**: creative עם system-prompt; סיווג/matching בלי (`withPersona: false`). matching מחזיר null בכשל — אין token-overlap fallback.
- **עטיפת preview נוצרת בתוך קריאת הדראפט/עריכה** (parsePreviewCopy). humanizer נפרד רק במסלול העריכה המקומית.
- **עברית קנונית; שינויים כירורגיים בלבד; Routing Trace על כל הודעה.**
- מזהי keyword מהודקים (אודיט 12.7) — לא לרופף בלי דוגמה חיה.
- **כרטיס היכולות (docs/karen-capability-card.md): כל שינוי במזהה מחייב עדכון של הכרטיס, ולהפך — אין להוסיף לכרטיס ניסוח שהקוד לא תופס.**

## שיטת עבודה (מחייבת)

1. אבחון והצעה בשפה פשוטה → אישור מפורש של איתן לפני כל קוד. **כל הצעה מתבססת על מה שקיים בקוד ומשפרת בוודאות — לא ממציאים ולא שוברים.**
2. שינוי כירורגי אחד בכל פעם. איתן מריץ הכל בעצמו ומדביק פלט.
3. העברת קוד: heredoc לקבצים חדשים; python עם אימות-לפני-נגיעה (ABORT) לקיימים. לא הורדות דפדפן. grep אחרי כל החלפה.
4. `npx tsc --noEmit` + QA ייעודי + רגרסיה → commit מדויק → בדיקה חיה. **אם tsc אדום — עוצרים, לא מקמטים.** הוראות מרובות-שלבים כוללות שערי עצירה מפורשים.
5. עדכון Handoff בסוף סשן. `git push` בסוף כל סשן.

## סבב UX 21.7 (מבוסס לוגים אמיתיים של קרן — הכל חי ומאומת)

**גשר בנק→גאנט (חי):** אחרי שמירת רעיון, אם יש תאריך פנוי — הצעה "לשבץ כבר או להשאיר בבנק?"; "לשבץ" משרשר אישור-להפקה + גאנט + שעה. משתמש ב-findSmartGanttDate.

חמישה שיפורים, כל אחד בקומיט נפרד עם QA ירוק:
- **עיצוב הודעות:** טיוטה = שם + סוג תוכן + כיוון בלבד (טון/עדיפות/קטגוריה ירדו). בנק = שמות בבולד + שורת רווח, בלי ID/קטגוריה/עדיפות.
- **קליטת שעה גמישה:** "ב-11:00" / "בשעה 11" / "ב11 ביום שישי" מתקבלים.
- **הזזת תאריך בגאנט:** "שנה תאריך של X ל-29/7" — detector מותנה-state, מזיז דרך updateGanttRowDate; תאריך תפוס → עצירה בטוחה + follow-up, בלי לדרוס.
- **חוקיות גאנט חכמה (findSmartGanttDate):** הצעת תאריך על רילים אורגניים בלבד — מקס' 2/שבוע + הפרדת יומיים ריל-לריל; סטורי/פוסט/שת"פ שקופים. מחובר לגשר; שאר נקודות ההצעה = פריט פתוח.
- **בנק→הפקה:** detector+extractor מורחבים לכל הנטיות; שם לא-קיים → רשימת בחירה במקום כפיל. פתוח: שם חלקי לכמה התאמות ("מכבי") נבחר שרירותית — לסבב הבא.
